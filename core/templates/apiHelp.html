API

It is possible to retrieve data in json format via curl so that it can be used programmatically. Every endpoint that is available in the web interface is also available via API, and the data is returned in json format. Do not use web page scraping; if you need info, ask how to get it.

There are several options for authentication:

1: Custom API Token Authentication

Use this method for custom Python scripts, command-line clients, or developer tools interacting with the APIs directly.

  1. Log in & go to the User Profile page (your name in top right corner -> Profile)
  2. You should see your token in the table. It has limited lifetime. If you need longer lived token, you may want to use a service account or go for option 2.
  3. Pass this token in the Authorization header of your requests prefixed with Token, example:
curl -X GET https://bigpanda.cern.ch/api/auth_test/ -H "Authorization: Token a1b2c3d4e5f6a8b9c1" -H "Accept: application/json"

2: IAM issued tokens

This relies on transient tokens generated via your local OIDC agent environment.

  1. See the documentation on how to set up OIDC agent with device flow
  2. Generate a token with openid, profile and wlcg scopes: oidc-token --scope="openid profile wlcg" YOUR-USER-DEVICE-FLOW-CLIENT
  3. Pass this token as a Bearer token inside the request's Authorization header

Example Request (cURL):
curl -X GET https://bigpanda.cern.ch/api/auth_test/ -H "Authorization: Bearer a1b2c3d4e5f6a8b9c1" -H "Accept: application/json"
